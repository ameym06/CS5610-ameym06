Introduction


Sound Bytes is my project under the course Web Developemnt (CS-5610) that I took during my graduation. This web application serves for Guest as well as Registered users. Both, guest and registered users can listen to music, watch youtube videos, find live concerts. The registered Users get the priviledge of writing blogs and viewing their blogs anytime they want.

APIs used for the project :

API USE
SoundCloud API This API is responsible for fetching the music / songs according to user queries
Youtube API This API is responsible for fetching the youtube vidoes according to the user queries
Eventful API This API is responsible for fetching the details related to upcoming events that are searched by user

Different Pages of Project :

PAGE DESCRIPTION
Home Page This is the Landing page. This page has links to all the services the web application provides. The user could log into the application to use additional services.
Login Page This is the Login page. A registered user can use this page to safely log into the web application
Sign Up Page This page can be used by a new / Guest user to create account. Using the newly created credentials, the user can then log into the application.
Sound Cloud Page This page is used for searching songs / music. This page invokes SoundCloud API to fetch results.
Youtube Page This page is used for searching youtube videos. This page invokes Youtube Data API (v3) to fetch results.
Concert Page This page is used for searching upcoming concerts of their favorite Artist(s). This page invokes Eventful API to fetch results.
Blog Page This page can be used by a Logged-In user to wite blogs. These blogs can be related to the experiences he / she had during any musical event / concert, etc. The user is free to blog about any topic.
My Blogs Page This Page fetches all the blogs written by the user from the database. It contains the links to view the entire blog
View Blog Page This Page is used to display a particular blog in details. The User can revisit any blog he / she wants anytime.

Technologies used:

TECHNOLOGY PURPOSE USED IN
HTML All the pages are designed using HTML Client Side
CSS / Bootstrap / Bootstrap-themes This is use to enhance the look and feel of project. Bootstarp also makes the pages responsive in nature Client Side
AngularJS AngularJS is extensively used technology throughout the project. It implements the MVC Model. Moreover, AngularJS routing is used to route across different pages. Angular Cookies is used to keep track of sessions. Client Side
jQuery JQuery is used very little in this project. It is used only to implement scrolling functionality through the Home Page Client Side
Java Script JS is used to render the results fetched from Youtube and SoundCloud onto their respective pages Client Side
Node.JS This technology is the backbone of server side functionality. It receives all the POST and GET calls from the client Server Side
Express (Node.JS Module) This helps in establishing web services on which the project work. Server Side
MongoDB This is the backend used for data storage purpose. Data Storage
Mongoose (Node module) Provides a way to programmatically interact with the data present in MongoDB data storage. Server Side
