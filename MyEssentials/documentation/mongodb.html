Data Storage - MongoDB


The UserData and the User Blogs' Data is handled in this project. For backend storage purpose MongoDB is used which is a no SQL database. In MongoDB there are collections and documents (JSON Objects).
The Collections required for this web application are:

Collection Name Purpose
UserSchema user This is the collection which stores data about the user i.e. the data to that user enters at time of registration
BlogSchema myBlogs TThis is the collection which stores data about the Blogs written by the user. It save the title, content and the timestamp of the Blog creation.
